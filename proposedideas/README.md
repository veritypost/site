# Proposed ideas — adult web reader

Four distinct design moves for veritypost.com that lean on restraint as
distinction. Each is self-contained: pick one and ship, pick all four
and they compose, pick none and park them.

All four tie back to the platform's thesis — **"commentary is earned, not given"** —
and exploit the fact that the business model (subscription + gated
comments) doesn't need ad-density home pages.

## The four ideas

| # | File | Strength |
|---|---|---|
| 1 | [01-sources-above-headline.md](01-sources-above-headline.md) | Trust signal visible first |
| 2 | [02-reading-receipt.md](02-reading-receipt.md) | Civic-feeling shareable proof of reading |
| 3 | [03-earned-chrome-comments.md](03-earned-chrome-comments.md) | **The product thesis rendered as UI** — other sites literally can't copy |
| 4 | [04-quiet-home-feed.md](04-quiet-home-feed.md) | Whisper-quiet home that reads like a magazine |

## The one to ship first

**#3 (earned chrome).** It's the only one of the four that competitors
can't imitate without abandoning their comment models. The receipt and
the quiet feed are aesthetic moves; earned chrome is architectural.

## Process

Each file describes:
- What the idea is
- Why it's different from what other news sites do
- Where in the codebase it lives (file paths + approximate line ranges)
- What changes to ship it
- What the trade-offs / risks are

No code has been written for any of these yet. Approval before build.
